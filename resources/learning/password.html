// hardening guide How to Strengthen Your Password

⬡ Use a Passphrase ▶
Chain 4 or more random unrelated words together: correct-horse-battery-staple. It's long, memorable, and has enormous entropy. Avoid famous phrases or song lyrics — use genuinely random words. A 5-word passphrase (~65 bits of entropy) is stronger than a short complex password.
Important
↕ Length Matters Most ▶
Every additional character exponentially multiplies the number of combinations an attacker must try. A 12-character password from a 94-char set has ~10²³ combinations. A 20-character password has ~10³⁹. Aim for 16+ characters minimum for anything critical.
Critical
◈ Mix All Four Types ▶
Using only lowercase gives a charset of 26. Add uppercase → 52. Add digits → 62. Add symbols → ~94. This multiplies the search space at every length. Don't just tack ! on the end — spread characters throughout. Avoid substituting @ for a or 3 for e; attackers use those rules too.
Important
⊘ Avoid Predictable Patterns ▶
Sequential runs like 123, abc, or qwerty are in every wordlist and rule set. Keyboard walks (zxcvbn, asdfgh) are checked first. Repeating characters like aaaa or 1111 collapse entropy dramatically. If your password follows a pattern a human invented, an attacker has already modeled it.
Critical
⊛ Never Reuse Passwords ▶
When a site is breached, attackers run credential stuffing — testing your leaked username/password pair across thousands of services automatically. A strong password reused across sites is effectively one breach away from everywhere being compromised. Use a unique password for every account, no exceptions.
Critical
>_ Use a Password Manager ▶
You only need to memorize one strong master password. The manager generates and stores unique, random, 20+ character passwords for every site. Good options: Bitwarden (open source, free), 1Password, KeePassXC (local only). Your browser's built-in manager is better than reusing passwords, but a dedicated manager is safer.
Useful
◉ Add a Second Factor (2FA) ▶
Even a strong password can be phished or stolen from a poorly secured server. TOTP apps (Aegis, Authy, Google Authenticator) add a time-based code a thief can't reuse. Better still: use a hardware key (YubiKey) which is phishing-resistant. SMS 2FA is better than nothing but vulnerable to SIM-swapping.
Important
◎ Check Breach Databases ▶
Run your email through haveibeenpwned.com to see if your credentials appeared in known breaches. Their password search checks if a specific password hash exists in breach data — without sending your password in plain text. If a password you use shows up, change it everywhere immediately.
